Frequently Asked Questions (FAQ)

1. How to submit a device complaint?
Go to the Complaints page, fill in the required information about the device that is faulty, describe the problem in detail, and submit the complaint. Make sure to include accurate information so the resolution can be fast.
2. How to edit an existing complaint?
Admins can edit complaints via the Data Entry page. Locate the complaint by ID or user name, update the fields necessary such as status, description, or assigned staff, then save the changes. This ensures data remains accurate and up-to-date.
3. How to approve a complaint?
Approval is done by selecting the status in the Data Entry page for the complaint. You can mark it as approved, pending, or rejected, depending on the evaluation of the reported issue.
4. How to print complaint reports?
Navigate to the Full Report page. Here, you can view all complaints, filter by device type or status, and click the print button. The system generates a printable version suitable for documentation or meetings.
5. Can admins add complaints on behalf of users?
Yes. Admins can go to the Data Entry page and submit complaints directly, which is useful if users cannot access the system or for bulk reporting of multiple devices.
6. How to search for complaints?
Use the search bar on the Complaints page. You can search by complaint ID, user name, device type, or keywords in the description. This helps quickly locate specific complaints.
7. How to track complaint status?
Complaint status is displayed in the Complaints page under the status column. Admins can update the status, and users can see it if the system is linked for notifications.
8. How to manage user accounts?
Admin accounts can be created, updated, or removed from the Sign Up and dashboard settings. Ensure only authorized personnel have admin access to maintain security.
9. How to reset a user password?
Currently, password reset is manual. Contact the main admin account to reset a user's password. The system may send an email confirmation if configured for security.
10. Can complaints be deleted?
Yes, only admins can delete complaints. Navigate to the complaint, verify it should be removed, and confirm deletion. This is irreversible, so proceed with caution.
11. How to filter complaints by device type?
On the Complaints page, use the filter dropdown to select the device type such as Printer, AC, or Projector. Only complaints matching the selected device type will be displayed.
12. Can admins export data?
Admins can export complaint data through the Full Report page. Data can usually be downloaded in PDF or CSV format for record-keeping and reporting.
13. How to sort complaints?
Complaints can be sorted by ID, user name, device type, or date submitted by clicking the column headers. Sorting helps identify urgent complaints quickly.
14. How to handle duplicate complaints?
Check for duplicate entries by user name, device type, and issue description. Admins can merge or close duplicate complaints to keep the database clean.
15. Can multiple admins manage complaints simultaneously?
Yes, the system allows multiple admins to log in and manage complaints at the same time. Ensure changes are coordinated to prevent conflicting updates.
16. How to view complaint history?
Select a complaint and click to view detailed information. All previous updates, approvals, and status changes will be shown for reference.
17. Can notifications be sent automatically?
Currently, notifications are manual. Admins may use the Notifications page to inform users about updates, approvals, or actions taken on their complaints.
18. How to ensure data accuracy?
Admins must carefully check the information entered for each complaint, verify user details, and ensure device information is correct before submitting or approving complaints.
19. How to logout safely?
Click the "Sign Out" button on the sidebar. This will end your session and redirect to the login page. Always logout when finished to prevent unauthorized access.
20. Who to contact for technical issues?
For any technical problems with the admin panel, contact the main system administrator. This includes login issues, data errors, or unexpected system behavior.
