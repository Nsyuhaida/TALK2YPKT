Frequently Asked Questions (Admin)

1. How does the complaint workflow operate?
Complaints are submitted by users and stored in the system as pending. Admins review the details, update the status, and take action until the complaint is resolved.
2. What statuses can be assigned to a complaint?
Complaints can be marked as Pending, Approved, In Progress, or Resolved. Status updates help track the current stage of each complaint.
3. Where can admins update complaint information?
Complaint details can be edited through the Data Entry page, including status updates, remarks, and assigned staff.
4. How is system performance measured?
System performance is calculated based on the number of resolved complaints compared to the total complaints recorded in the system.
5. Can multiple admins work at the same time?
Yes. The system supports multiple admins accessing and managing complaints simultaneously with real-time data updates.
6. How can admins identify urgent complaints?
Admins can use filtering and sorting features to prioritize complaints based on date submitted, device type, or current status.
7. Is complaint history stored in the system?
Yes. Every complaint keeps a record of previous updates, status changes, and actions taken for reference and auditing purposes.
8. Can admins generate reports from the system?
Admins can generate and print reports from the Full Report page for documentation, meetings, or management review.
9. What happens if incorrect data is entered?
Admins can correct incorrect entries by editing the complaint details to ensure accurate and reliable system records.
10. How should admins end a session properly?
Admins should always click the Sign Out button after completing tasks to protect system data and prevent unauthorized access.
